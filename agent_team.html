How it works

どうやって作るの？

技術的な裏側を、ふんわり解説

🌷 仕組みは「指示書」の使い分け 🌷

1
各エージェントごとに「CLAUDE.md」という指示書ファイルを用意
2
その中に「あなたは○○担当」「こんな時はこうする」を細かく書く
3
Claude Codeでそのフォルダに入ると、自動でその子が起動する仕組み🌸
📄 hoshi/CLAUDE.md # SNS担当エージェント・ほし

## あなたの役割
あなたは 「ほし」 という
SNS発信担当のAIエージェントです。

## 担当領域
フォロワーさん向けの
日常発信・お役立ち投稿

## トーン
やさしく、共感的、押し付けない🌷

## やってはいけないこと
- 誇張表現
- 他の人の悪口
- 個人情報の特定につながる発信
📌 大事なポイント
各CLAUDE.mdは「育成手帳」みたいなもの🌷
最初は数行でも、使いながら少しずつ書き足していくと、その子がどんどん賢くなります✨

How to run

どうやって動かすの？

2つのスタイルがあります🎀
🏃‍♀️ 👯‍♀️ 2つのスタイル

① バトンリレー型（順番に渡す）
② 同時マルチ型（並列で動かす）

どっちもメリットがあるので
場面ごとに使い分けるのがコツ🌸

🏃‍♀️ ①バトンリレー型
ひとりずつ順番に仕事を渡していく
🍑
もも（人事）
「今週の企画はAの方向で」と方針決定
↓ バトンを渡す
🗺️
なぎ（戦略）
「具体的にこの順で進めよう」と設計
↓
📐

↓
📸
ほし（SNS）
完成品をもとに発信用に展開
✿ メリット
  • 前の人の成果を踏まえて次が動くので品質が安定
  • 途中で「ストップ」がかけられる安心感
  • 使うエージェントが1人ずつなので料金が読みやすい
  • 非エンジニアでも感覚的に運用できる
⚠ デメリット・留意点
  • 時間がかかる。前が終わらないと次に進めない
  • 前のエージェントの出力に依存するので、最初がコケると全体が崩れる
  • 引き継ぎ時に情報が抜け落ちることがある（人間のリレーと同じ）
  • 毎回ママが「次は誰？」と司令を出す手間が発生
🎀 こんなときに向いてます
企画→制作→校正→発信、のような順序が重要な仕事
最終アウトプットの品質を担保したいとき
初心者で、まず1本キレイに通したいとき

Environment

並列で動かすには
何が必要？

環境構築のおはなし🌷
🛠️ ざっくりまとめると

並列処理には4つの道具が必要🌸

① Claude Code（本体）
② VSCode（操縦席）
③ Git worktree（作業場の分離）
④ tmux / ccmanager（複数同時操作）

🤖
① Claude Code
エージェント本体
これがないと始まらない主役🌟
ターミナルで動くAIエージェントで、最近のバージョンから並列実行が標準サポートに。
💡 ポイント バージョン2.1.49以降は
--worktree オプションで簡単に並列起動できるようになりました✨
必須 npm install Pro以上推奨
🖥️
② VSCode
エディタ・操縦席
Microsoftが作った無料のエディタ。
ファイル編集・ターミナル・拡張機能ぜんぶ入りで、司令室のような役割を果たします🌷
💡 並列処理での活躍ポイント ・複数ターミナルを画面分割で開ける
・各worktreeの様子を一目で監視
・Claude Code拡張機能でさらに快適に
無料 Win/Mac対応 拡張機能豊富
🌳
③ Git worktree
作業場を分ける魔法
ひとつのプロジェクトを複数の作業フォルダに分身させる技🌟
これがないと、エージェント同士がぶつかってケンカします💦
💡 イメージで例えると キッチンが1つだと料理を1品ずつしか作れないけど、
分身キッチンを4つ作ると、4品同時に料理できる🍳
（冷蔵庫=git本体は共有のまま）
Git標準機能 独立環境 並列必須
🎛️
④ tmux または ccmanager
複数セッション管理ツール
複数のエージェントを一画面で同時監視できるツール。
どっちか好きな方でOK🌷
🌸 tmux 昔からあるターミナル分割ツール。
Claude Codeに --tmux オプションで連携できる

🌷 ccmanager Claude Code用に作られた専用ツール。
tmuxよりカンタンで初心者向け✨
Gemini / Codex などにも対応
画面分割 同時監視 どちらか1つ

# 各キッチン = git worktree
# 画面分割 = tmux または ccmanager
# 司令塔 = ママのVSCode 🎀

🎀 並列処理スターターパック 🎀

1
Mac または Windows PC メモリ16GB以上推奨（並列で重くなる）
2
Node.js のインストール Claude Codeを動かすのに必要
3
VSCode のインストール 公式サイトから無料ダウンロード
4
Claude Code のインストール Pro以上のサブスク + npm install
5
Git のセットアップ プロジェクトをgit管理にしておく
6
ccmanager または tmux npxですぐ試せるccmanagerが初心者◎

⚠️ 環境構築で気をつけたいこと

  • 🌸 PCのスペック：4並列ならメモリ16GB、6並列以上なら32GB推奨
  • 🌸 料金プラン：並列実行はMax plan ($100/月)以上がオススメ。Proだと上限にすぐ達します
  • 🌸 worktreeの整理：放置すると増えすぎてカオスに。定期的にお掃除を
  • 🌸 同じファイルの編集競合：別worktreeでも結局1つのプロジェクトに統合するので、最後にコンフリクト解決が必要
  • 🌸 ネット環境：複数同時にAPI叩くので、Wi-Fi弱いとつらい💦

Start Guide

あなたも作ってみよう🎀

小さく始めるのがコツ

🌷 3ステップではじめる 🌷

STEP 1: まず1人だけ作る
いきなり9人は無理！
まずは「献立担当」「保活担当」など、一番困ってる領域の1人から始めましょう🌸
STEP 2: 名前と性格を決める
例：「ミルキー（離乳食担当・1〜3歳の専門知識・優しい口調）」
名前があるだけで愛着UPします💕
STEP 3: 使いながら育てる
最初は不完全でOK。「もうちょっと簡潔に」「アレルギーは絶対チェックして」など毎回フィードバックして、指示書を更新していけば賢くなります✨

③ NGリストを書く 「やってはいけないこと」が重要
